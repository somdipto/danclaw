## Deploy Wizard Review

### Current Flow
1. Select Model → 2. Select Channel → 3. Configure → Deploying

### Issues Found
1. **No inline validation** - User can click "Next" with invalid input
2. **No price preview** - User doesn't see estimated cost until after deploy
3. **Error state loses config** - If deployment fails, all settings are lost

### Recommended Improvements
```
1. Add validation message: "Please select a model to continue"
2. Show price badge: "~$29/mo" next to tier selection
3. Preserve form state on error - add retry without re-entering
```

## Chat Page Review

### Current Implementation
- Deployment selector dropdown in header
- Message bubbles with user (right) / agent (left) alignment
- Typing indicator with bouncing dots
- File attach + voice buttons (non-functional)

### Issues Found
1. **No message actions** - Can't copy/edit agent responses
2. **Empty state is good** but could link directly to deploy
3. **WebSocket status indicator** is subtle (2px dot)

### Recommended Improvements
```
1. Add hover menu on messages: Copy | Edit | Delete
2. Change empty state CTA: "Deploy Agent" → opens deploy wizard
3. Make WS indicator more prominent or add "Reconnecting..." state
```

## Mobile vs Web Consistency

### Theme Colors
| Element | Web (Tailwind) | Mobile (React Native) |
|---------|---------------|----------------------|
| Primary | `#6366F1` | `#6366f1` ✅ |
| Background | `#0A0A0F` | `#0C0C14` ⚠️ Slightly different |
| Card | `#1F2937/50` | `#111118` ⚠️ Different |

### Component Patterns
- Web: Tailwind classes directly
- Mobile: `ThemedView`, `ThemedText` with theme hooks
- Status: **Adequate consistency** - minor color values differ
