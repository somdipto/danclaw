## Deploy Wizard — 3 Specific Improvements

### 1. Add Step Summary Before Deploy
**Issue**: The config step shows all selections but there's no price estimation or confirmation before hitting "Deploy Agent".

**Fix**: Add a summary panel with:
- Monthly cost estimate based on selected tier
- Estimated API costs
- Final confirmation before mutation

```tsx
// Add to Config step before the final button row
<div className="p-4 rounded-xl bg-dark-800/50 border border-dark-700/50">
  <div className="flex justify-between text-sm">
    <span className="text-dark-400">Monthly estimate</span>
    <span className="text-white font-medium">$29/mo</span>
  </div>
  <p className="text-xs text-dark-500 mt-2">Final costs calculated on usage</p>
</div>
```

### 2. Visual Feedback on Form Validation
**Issue**: The Next buttons disable without explaining WHY they're disabled.

**Fix**: Add inline validation hints below inputs:
```tsx
{!serviceName && (
  <p className="text-xs text-dark-500 mt-1">Enter an agent name to continue</p>
)}
```

### 3. Persist Wizard State on Refresh
**Issue**: Refreshing the page loses all selections.

**Fix**: Use URL params or localStorage to persist step and selections.

## Chat Page — 3 Specific Improvements

### 1. Add Connection Status Indicator with Retry
**Issue**: WS disconnected state just shows a gray dot. No auto-reconnect or user feedback.

**Fix**: Add explicit connection status with retry button:
```tsx
<div className={`w-2 h-2 rounded-full ${
  wsConnected ? 'bg-secondary-500' : 'bg-red-500'
}`} title={wsConnected ? 'Connected' : 'Disconnected'} />
{!wsConnected && (
  <Button size="sm" variant="ghost" onClick={reconnect}>Reconnect</Button>
)}
```

### 2. Add Message Copy Action
**Issue**: Users can't copy agent responses.

**Fix**: Add hover copy button on messages:
```tsx
<button 
  onClick={() => navigator.clipboard.writeText(msg.content)}
  className="opacity-0 group-hover:opacity-100 text-dark-400 hover:text-white"
>
  📋
</button>
```

### 3. Debounce Input for Better UX
**Issue**: Rapid Enter keystrokes can queue multiple sends.

**Fix**: Disable send briefly after submission.
