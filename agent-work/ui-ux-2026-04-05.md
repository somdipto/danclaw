## Deploy Wizard Review

**File:** `apps/web/src/app/dashboard/deploy/page.tsx`

### ✅ What's Working Well
- Clean 3-step wizard flow (Model → Channel → Config)
- Progress indicator shows completed/current/pending states clearly
- Model and channel cards show good detail (icon, name, provider, description)
- Disabled state for unavailable channels (Telegram shows "Soon" badge)
- Error banner with dismiss button

### 🔧 Improvements (3 Specific)

**1. Tier Selection is buried in Config step — needs visual hierarchy boost**

Currently the tier selector (Free/Pro/Elite) is just a row of buttons at medium text size. For pricing decisions, this needs more visual weight.

```tsx
// Current — buried in config
<div className="grid grid-cols-3 gap-3">
  {PRICING_TIERS.map((tier) => (
    <button key={tier.tier} ...>
      <p className="font-semibold text-white">{tier.name}</p>
      <p className="text-xs text-dark-400 mt-1">{tier.priceLabel}</p>
    </button>
  ))}
</div>
```

**Suggested fix:** Add tier cards with feature highlights, or at minimum make them taller cards with a "recommended" badge on the middle tier. Consider adding a tooltip explaining what each tier includes.

**2. No way to go back to previous steps without losing state**

The wizard works fine but there's no visual indication that clicking "← Back" preserves the already-selected model/channel. The back button is present (variant="ghost") but could use better visibility in the Config step.

**Suggested fix:** The back button is already there in the Config step. This is fine.

**3. Missing inline validation for service name**

The service name field normalizes input (lowercase, hyphens) but doesn't validate uniqueness or show if a name is taken until after attempting deploy.

**Suggested fix:** Add a debounced availability check (e.g., after 500ms of no typing, check if service name is available) with a green checkmark or red X indicator.

## Chat Page Review

**File:** `apps/web/src/app/dashboard/chat/page.tsx`

### ✅ What's Working Well
- Empty state with clear CTA to deploy
- Typing indicator with bounce animation
- WebSocket connection status dot
- Auto-scroll to latest message
- Enter-to-send (without shift), Shift+Enter for newlines
- Deployment selector dropdown

### 🔧 Improvements (3 Specific)

**1. WebSocket status indicator is too subtle**

The WS connection dot (🟢/⚫) is small (w-2 h-2) and positioned far right in the header. If the connection drops, users may not notice.

```tsx
// Current — hard to spot
<div className={`w-2 h-2 rounded-full ${wsConnected ? 'bg-secondary-500' : 'bg-dark-600'}`} />
```

**Suggested fix:** Move connection status to a more prominent position or add a toast notification when connection is lost/re-established. Consider showing "Connecting..." text state.

**2. No visual distinction between message types**

All messages render as text in bubbles. If the agent sends structured data (JSON, cards, etc.) or images, there's no differentiation.

**Suggested fix:** Add a message type system that renders different content appropriately:
- `type: 'response'` — standard text bubble
- `type: 'card'` — renders structured card UI
- `type: 'image'` — renders inline image
- `type: 'error'` — renders error message with red styling

**3. No "retry" for failed messages**

If a user sends a message and it fails (WS disconnected), there's no visual indication the message was not delivered, and no retry button.

**Suggested fix:** When a message fails to send, add a small error icon and "Tap to retry" on the failed message bubble.

## Empty States & Loading States Pattern

### ✅ Good Patterns Found

**Dashboard Empty State:**
```tsx
<Card className="text-center py-12">
  <p className="text-4xl mb-4">🚀</p>
  <p className="text-white font-medium mb-2">No agents yet</p>
  <p className="text-dark-400 text-sm mb-6">Deploy your first AI agent in under 60 seconds</p>
  <Link href="/dashboard/deploy">
    <Button icon={<span>🚀</span>}>Deploy Your First Agent</Button>
  </Link>
</Card>
```

**Chat Empty State:**
```tsx
<div className="flex flex-col items-center justify-center h-full">
  <p className="text-4xl mb-4">💬</p>
  <p className="text-white font-medium mb-2">No running agents</p>
  <p className="text-dark-400 text-sm mb-6">Deploy an agent to start chatting</p>
</div>
```

### 📋 Pattern to Document

All empty states should follow this structure:
1. **Contextual emoji** (4xl) — sets emotional context
2. **Title** (white, font-medium) — what is missing
3. **Description** (dark-400, text-sm) — why it matters / what to do
4. **Primary CTA** (Button) — single clear action
5. **Optional secondary** — if there's a secondary path

## Mobile App Consistency Review

**Files reviewed:**
- `apps/mobile/src/constants/theme.ts`
- `apps/mobile/app/(tabs)/deploy.tsx`
- `apps/mobile/app/(tabs)/chat/index.tsx`
- `packages/shared/src/theme/index.ts`

### ✅ Aligned Areas
- Color hex values match (primary: #6366F1, secondary: #10B981, etc.)
- Dark theme base colors aligned (dark-950: #0A0A0F vs mobile: #0C0C14 — minor 0.3% difference, acceptable)
- Component structure similar (Card → TouchableOpacity with card style)

### ⚠️ Inconsistencies Found

**1. Spacing scale mismatch**

Web uses Tailwind's 4px-based scale (space-1 = 4px, space-2 = 8px, etc.)
Mobile theme.ts uses custom Spacing constants (Spacing.two = 8, Spacing.three = 16, etc.)

This means spacing decisions must be made manually. Consider aligning more closely or documenting the mapping.

**2. Mobile lacks Button component**

Mobile uses inline `TouchableOpacity` + text styling instead of a shared Button component. This leads to:
- Inconsistent border-radius (mobile uses 14 for buttons, web uses 12 for `rounded-xl`)
- No shared loading state component
- No shared icon prop

**Suggested fix:** Create a shared `MobileButton` component that matches the web Button variants.

**3. Mobile Deploy wizard missing tier selection**

The web wizard has a tier/pricing selection step. The mobile wizard hardcodes `tier: 'free'` with no UI to change it.

```tsx
// Mobile hardcoded
const req: CreateDeploymentRequest = {
  service_name: serviceName.trim(),
  tier: 'free',  // <-- hardcoded
  region: selectedRegion,
  model: selectedModel,
  channel: selectedChannel,
};
```

**Suggested fix:** Add tier selector UI to mobile deploy wizard matching the web.

**4. Status dot animation missing on mobile**

Web Badge has `animate-pulse` on active statuses. Mobile `StatusDot` is static.

```tsx
// Web Badge.tsx
dot: 'bg-secondary-500 animate-pulse-slow',  // pulsing

// Mobile StatusDot
const color = status === 'running' ? Colors.secondary : ...;
return <View style={[styles.statusDot, { backgroundColor: color }]} />;  // static
```

**Suggested fix:** Add `animate-pulse` equivalent using React Native's `Animated` API or `expo-task-manager` if available.
