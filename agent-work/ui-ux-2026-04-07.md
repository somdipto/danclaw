# UI/UX Design Review — 2026-04-07

## Executive Summary

Reviewed all Phase 1 pages (deploy wizard, chat, dashboard) and UI components. Design system is well-documented in `DESIGN.md` and `COMPONENTS.md`. Key findings: design language is consistent across web components, but the chat page needs immediate UX fixes (missing CTA, no typing indicator). Mobile has minor color inconsistencies that should be aligned.

---

## Pages Reviewed

### 1. Deploy Wizard (`apps/web/src/app/dashboard/deploy/page.tsx`)

**What works:**
- 3-step flow with animated step indicator
- Staggered animations via Framer Motion
- Glass card aesthetic consistent with design system
- AnimatePresence for smooth step transitions

**Issues & Improvements:**

| # | Issue | Severity | Recommendation |
|---|-------|----------|----------------|
| 1 | Step state is local React state — lost on refresh | Low | Add URL params (`?step=1`) for bookmark/share support |
| 2 | Model cards show cramped description text | Low | Add tooltip on hover/focus with expanded model details |
| 3 | Step 2 summary has no way to edit model/channel inline | Low | Add edit icons next to summary items that jump to correct step |

**Quick wins:** #1 and #3 are <30 min each.

---

### 2. Chat Page (`apps/web/src/app/dashboard/chat/page.tsx`)

**What works:**
- Clean message bubbles with user/agent differentiation
- Deployment selector for multi-agent switching
- Real-time WebSocket integration
- Good empty state with emoji

**Issues & Improvements:**

| # | Issue | Severity | Recommendation |
|---|-------|----------|----------------|
| 1 | Empty state ("No running agents") has no CTA button | High | Add `<Button onClick={() => router.push('/dashboard/deploy')}>Deploy Agent</Button>` |
| 2 | No typing/thinking indicator when agent is processing | Medium | Add 3-dot bounce animation bubble after user sends message |
| 3 | Timestamps show time only — loses date context on history | Low | Format: "Today 2:30 PM", "Yesterday 11:00 AM", "Apr 4, 2:30 PM" |

**What works:**
- Stats cards with trends and icons

## Component Library Review

### `Badge.tsx`
- Status config uses semantic colors (secondary/primary/accent)
- Missing `animate-pulse-slow` for "running" status — inconsistent with other pulsing statuses

### `Button.tsx`
- 5 variants (primary, secondary, ghost, danger, outline) — well structured
- Loading spinner included
- `active:scale-[0.98]` for tactile press feedback

### `Card.tsx`
- Glass effect: `bg-dark-800/50 backdrop-blur-xl border border-dark-700/50`
- Hover prop adds lift effect
- `glow` prop for primary glow (not used in current pages)

### `StatsCard.tsx`
- Clean layout: icon + trend badge top, value (3xl bold), label (sm muted)
- Trend uses secondary-500 for up, red-500 for down

### `ProgressBar.tsx`
- 3 colors (primary, secondary, accent)
- Glow effect on bar
- Clamps 0-100
